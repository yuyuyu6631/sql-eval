

# Text-to-SQL 自动化测试平台 PRD (最终版)

## 一、 产品概述

本平台旨在客观、高效地评估大语言模型或智能体的 Text-to-SQL 能力。通过自动生成基准用例、在隔离沙箱中真实执行 SQL 并比对结果集，辅以高阶大模型的诊断分析，为开发人员和测试人员提供一套“白盒化”的评测与调试工作台。

## 二、 核心重构业务流程（The "Right" Way）

1. **基准构建 (Baseline Generation)：** 平台输入 DDL，调用基础大模型生成 `[自然语言问题, 标准SQL]`。后端在沙箱中**强行试跑标准 SQL**，剔除语法错误或查无数据的废料，沉淀为高质量测试集。
2. **盲测获取 (Blind Testing)：** 平台仅将 `[自然语言问题]` 和 `[DDL]` 发送给待测智能体，获取其生成的 `[测试SQL]`。（绝不泄露标准 SQL）。
3. **安全拦截 (Security Filter)：** 平台后端拦截智能体 SQL 中的 `DROP/DELETE/UPDATE` 等高危指令。
4. **沙箱双跑 (Execution & Data Diff)：** 将 `[标准SQL]` 和 `[测试SQL]` 同时扔进只读沙箱执行。对比两者的**最终数据返回结果**是否一致。一致即 100% Pass。
5. **高阶兜底 (LLM Judge & Diagnosis)：** 仅当结果集不一致或执行报错时，调用高阶大模型（如 GPT-4 / Gemini Advanced）。将其作为“裁判”分析错因（如：选错表、漏条件、聚合错误），而非简单打分。

## 三、 详细功能需求说明

### 3.1 模块一：资产与环境配置管理

* **目标：** 管理测试物料和外部接口接入。
* **功能点：**
* **数据源绑定：** 配置沙箱数据库连接。**强制要求开启只读模式（Read-Only）或事务回滚机制**。
* **表结构（Schema）管理：** 支持一键导入 DDL，支持手动添加关键字段的枚举值注释（极大提升智能体准确率）。
* **智能体管理：** 配置待测智能体的 API 地址、鉴权 Token、请求超时阈值及并发限制。



### 3.2 模块二：基准测试集工厂

* **目标：** 自动化生产并清洗测试用例。
* **功能点：**
* **一键生成：** 选择特定的表结构，配置生成数量（如 100 条）。
* **自清洗漏斗展示：** UI 需清晰展示生成过程的漏斗：`请求生成(100) -> 语法正确(90) -> 有效数据(85) -> 最终入库(85)`。
* **用例管理列表：** 展示 `[问题, 标准SQL]`。支持测试人员手动增删改，支持对复杂 SQL 添加业务标签（如 `多表JOIN`, `时间聚合`）。



### 3.3 模块三：自动化评测执行引擎 (核心后台)

* **目标：** 高效、安全地执行批量测试。
* **功能点：**
* **任务编排：** 选择测试集 + 选择智能体 + 选择高阶诊断模型（可选） = 发起测试任务。
* **限流与重试并发控制：** 后端必须支持对智能体 API 的限流（Rate Limit）和失败指数退避重试，防止将待测服务打挂。
* **强制安全限制：** 所有执行的 SQL 强制追加 `LIMIT 100`，强制设定查询超时时间（如 5s），超时按 Fail 记并记录 Timeout 错误。



### 3.4 模块四：诊断与分析工作台 (UI 核心痛点解决)

* **目标：** 提供深度的错误排查体验，拒绝只看通过率。
* **功能点：**
* **宏观看板 (Dashboard)：**
* 核心指标：真实执行通过率、语法错误率、超时率。
* 下钻图表：按 SQL 复杂度分类的失败率分布；高频错因分布（饼图展示大模型诊断出的错因分类）。


* **微观诊断调试台 (Diff UI)：**
* **结构化 SQL 差异对比：** 采用专业代码高亮组件（VueMonacoEditor），左右分屏（Split Panel）高亮只读展示 `标准SQL` (左) 与 `智能体SQL` (右) 的原生写法差异。
* **结果集比对预览：** 在 SQL 代码编辑器的正下方，继续左右分屏展示沙箱中真实查出的 `预期结果集Table` (左) 和 `实际结果集Table` (右)，直观呈现数据层面的缺失与条数对比。
* **AI 诊断面板：** 在页面的最底部展示高阶大模型生成的具体错因报告（必须支持由 Markdown 引擎如 `marked` 解析的结构化呈现：包含错误类型、根本原因、差异对比与修复建议等大模型产出文本）。

## 四、 关键数据库表结构提要 (支持上述流程)

* `schema_env`: 管理 DDL 和沙箱 DB 凭证。
* `test_case`: 存储清洗后的 `[问题, 预期SQL]`。
* `agent_config`: 待测智能体接口配置。
* `evaluation_task`: 记录一次批量测试任务的整体状态和进度。
* `evaluation_result` (最核心):
* `agent_sql`: 智能体返回的 SQL。
* `execution_status`: 执行状态 (Success / Syntax Error / Timeout / Security Blocked)。
* `data_diff_passed`: 结果集比对是否一致 (Boolean)。
* `ai_diagnosis`: 高阶模型给出的错因定性分析文本。
