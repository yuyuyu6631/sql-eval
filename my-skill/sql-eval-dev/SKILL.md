# SQL 自动化评测平台开发专用 Skill

本技能专用于保障 `d:\数璇项目开发\file\` 目录下的“SQL 自动化评测平台”在开发或 Bug 修复时，严丝合缝地遵循专属的开发指南、架构演进目标及代码质量红线。

---

## 执行步骤

### 第一步：对齐项目总控指南
任何涉及业务逻辑的修改或新增前，必须优先阅读并参照 `d:\数璇项目开发\file\AGENT_DEVELOPMENT_GUIDE.md`。
阅读后，明确用户需求属于 TASK-01 到 TASK-07 中的哪个环节，或是否符合现有技术栈标准。

### 第二步：对照任务指令与验收标准落地
- 定位修改位置：严格按照开发指南所指示的具体文件路径和代码行进行，不可自行猜测。
- 保证安全红线：若需求涉及高优任务（如并发执行、API外部接入），需保证加上降级通道（Mock）、设定合理超时和失败重试机制（如 `tenacity`），防止局部雪崩影响主体服务。
- 依赖同频：若引入了新包（如 `asyncpg` 或 `@guolao/vue-monaco-editor`），立即同步更新 `requirements.txt` 或 `package.json`。

### 第三步：输出与质量检查
修改完成后，必须自我校验是否达成了指南中该 TASK 的“验收标准”。

---

## 输出格式

1. **全面中文化**：过程的思考 Plan、任务核对 Task 及给用户的回复，**必须是纯中文**。
2. **本土化注释**：新写的代码或做出的逻辑改动，需加上地道中文注释说明背景（例：`# 使用 Semaphore 控制并发，防止打挂被测服务`）。
3. **闭环交付说明**：告知用户变更内容，且给出验证方式或如何使用测试脚本（如提示 `npm run dev` 或 `start.bat` 观察日志）。

---

## 注意事项

- 必须基于绝对路径 `d:\数璇项目开发\file\` 工作。
- 不要自作聪明地用一行代码带过复杂修改，严格按指令实施细节执行。
- 遇到当前架构和文档未能囊括的问题时，先提示用户设计空缺并给出方案，等待确认后再开发。
